# Golden Paths — one-command deploy per agent

Every agent ships a **SAM golden path** that deploys the *real* agent end to end: a shared layer (platform_core + governance + the agent's `core.py`), the five workflow Lambdas (classify -> draft -> check -> **human gate** via `waitForTaskToken` -> finalize), the governed connector, the real Step Functions ASL, an HTTP API with a **Cognito JWT authorizer + access logging + throttling**, **per-function least-privilege roles** (Bedrock scoped to model+guardrail ARNs), hardened Cognito (MFA + immutable role + short tokens), a Bedrock Guardrail (prompt-attack in+out), and an audit table. Each agent's **Bedrock Guardrail is tuned to its data class** (e.g. CJI identifiers for public safety, FTI/PHI for benefits, credentials/secrets for the IT desk), and its smoke test approves with a **domain-appropriate reviewer role**. All cfn-lint clean. **Maturity:** these are governed, deployable **workflow skeletons** — the HTTP tool route is enforced through the gateway (policy/approval/token/append-only audit); **all 8 workflows now run their consequential write through the gateway** (policy + bound approval + scoped token + append-only audit). Every workflow now runs Classify → **Retrieve** (governed `kb.search_policy` read through the gateway, audited) → **Draft** (Bedrock Converse + the deployed Guardrail) → **Check** (grounding + output guardrail) → human gate → governed write, with a **deterministic fallback** so the suite runs offline. The **human gate is served by a real reviewer service** (P7): a reviewer authenticates to `GET /approvals` and `POST /approvals/{id}/decision`; the service checks authority + separation of duties, mints the bound token server-side, resumes the execution, and audits the decision (`mint_approval.py` remains only as an offline smoke stand-in). **Every agent now ships a secure combined deploy** — `infra/golden-path-*-secure/` provisions the secure baseline and the app in one `sam deploy` (isolated VPC + PrivateLink endpoints + customer-managed CMK + S3 Object-Lock WORM + CloudFront/WAF, Lambdas in-VPC, tables/logs CMK-encrypted; distinct VPC CIDRs 10.40–10.47/16). These are the production-shaped counterparts to the base skeletons (deploy in us-east-1 for the CloudFront-scoped WAF). Live system-of-record connectors and a cloud integration-test pipeline are the remaining tracked delivery phase (`docs/REVIEW-2-IMPROVEMENTS-BACKLOG.md`).
